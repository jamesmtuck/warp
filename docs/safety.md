# Warp Safety Philosophy

## Core Principle

**Warp never auto-executes generated commands.**

All commands generated by Warp are presented as drafts for your review. You explicitly choose to insert a command into your shell buffer, and you press Enter to run it — Warp never does this for you.

## Deterministic Safety Layer

Every generated command passes through a deterministic safety analyzer (`safety.py`) before being shown to you, regardless of which AI backend is used.

### Risk Levels

| Level | Examples |
|-------|---------|
| **low** | `ls -la`, `git status`, `echo hello` |
| **moderate** | `rm file.txt`, `sudo apt install`, `chmod -R`, `kill -9`, `> file` redirect |
| **high** | `rm -rf`, `find -delete`, `dd`, `mkfs`, `rm *.tmp` |

### Safer Previews

For high-risk commands, Warp suggests a safer preview variant:

| Risky | Safer Preview |
|-------|--------------|
| `find . -name "*.log" -delete` | `find . -name "*.log" -print` |
| `rm *.tmp` | `printf '%s\n' *.tmp` |
| `rm -rf /path` | `echo rm -rf /path` |

## Why Deterministic Safety?

AI models can hallucinate or produce subtly dangerous commands. The safety layer is rule-based and deterministic — it doesn't rely on the AI to self-report risk. It runs on every candidate, every time.

## Shell Integration Safety

Shell hooks (`preexec`/`precmd`) are designed to:
- Never block your shell
- Fail silently if warp is unavailable
- Run capture in a background process
- Never modify your current command or environment

## What Warp Does NOT Do

- Does NOT execute any command automatically
- Does NOT modify your shell history directly
- Does NOT send your history to any remote service (unless you configure the openai backend)
- Does NOT run shell hooks that can break your session
